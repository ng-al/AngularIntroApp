Custom Validators

Custom validators in AngularJS are directives. They can be synchronous or asynchronous. Asynchronous validators are used when you need to call the server to perform the validation test. For example, in this application when adding or editing a user, we want the email to be unique. The only way to test this, is to ask the server to make this test via a database query.

Synchronous Validators

Within the Routing/Dynamic section, there is a validator (/topics/routing/dynamic/properUriValidator.js) that ensures that the typed in path begins with a slash and does not contain a hash.

Asynchronous Validators

Within the CRUD/Edit and CRUD/Add sections, there is a validator (/topics/user/emailUniqueValidator.js) that ensures (via a server call) that two users do not have the same email address.
