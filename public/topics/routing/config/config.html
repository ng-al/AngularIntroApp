Routing Configuration (with ngRoute)

When an AngularJS application starts; before directives and controllers referenced by the DOM are created, the Angular framework executes the configuration and run blocks for all of the modules.

Configuration blocks are defined by angular.module("myModuleName").config(...); All of the configuration blocks (in all injected modules) are executed. The purpose of configuration blocks are to initialize application settings. Within a config block you can access value recipes and providers, but you do not have access to services since they are constructed during run time.

ngRoute requires the application to define routing data using the $routeProvider during configuration time. The routing data is a mapping between URIs and their corresponding templates (and controllers).

For example, the page you are reading uses this declaration:

$routeProvider
    .when("/routing/Config", {
        templateUrl: "topics/routing/config/config.html"
    });

The source of the routing configuration can be found at /app/routing/routeConfig.js
