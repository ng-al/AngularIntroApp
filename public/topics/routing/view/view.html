ngRoute Views

When the URI changes:

  • By typing in the browser's address bar, or
  • Clicking on a link, or
  • programmatically changing the path using the $location service

The router attempts to find the URI in the set of routes defined during configuration. If it finds the URI, it will render the template.

Where does the router insert the template? That is the purpose of the ng-view directive.

If you examine the index page (/server/views/index.ejs) you will see a line like this:

<div ng-view><div> (On line 63)

This marks the place in the DOM where the router will render the templates after a change in the path.

Note that ngRoute only supports a single ng-view directive. The other routers support multiple view insertion points.
