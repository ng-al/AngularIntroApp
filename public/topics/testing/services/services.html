Unit Testing Services

Getting the service (to test) is easy: You just inject it. However, to properly unit test a service, you must isolate it from the dependent services it injects. This is done via $httpBackend and mocking Testing/Mocking.

When not to Test

If all your service does is proxy server-side REST requests via $http, then there is NO need to test that service at all since there is no logic within the service to test. Services that contain logic (helper functions, utilities, etc.) should be tested.

The unit test that is provided in this sample app is an example of a test that should not have been written. It is only here to demonstrate mocking a REST API via $httpBackend.

$httpBackend

When an Angular application needs some data from a server, it calls the $http service, which sends the request to a real server via the $httpBackend proxy. With dependency injection, you use the $httpBackend service (from ngMock which has the same API as $httpBackend) and use it to verify the requests and respond with test data without making actual HTTP requests.
