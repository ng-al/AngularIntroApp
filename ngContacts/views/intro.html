Contacts Application

Introduction

This small application is designed for you to change. While the Contact List page is functional, the Create, Update, and Delete pages are not. In addition, the app uses essentially no styling.

Instructions to the Javascript Applicant

There is a mock repository service that simulates a simple database. It is initialized when the app starts up and is used by the contactListCtrl. Your goal is to take the three non-functioning pages and make them work. To do this you need to create a service AS IF the data was provided over HTTP from a remote server.

Although the Contact List page inerfaces with the mock repository directly, the other three pages should NOT inject MockRepository; rather they will consume a "contactService" that you will create.

  1. Create a "contactService".
    • The service should provide: createContact, updateContact, and deleteContact.
    • Internally (in the service), use the Mock Repository to perform the actual work.
    • The create and update methods should return the new/updated contact object (the repository returns this to you.)
    • Like a "real" service, your service should not return the objects directly; use promises. See note, below.
  2. Finish the Create Contact page.
  3. Finish the Update Contact page.
  4. Finish the Delete Contact page.
Note: The promise that your service returns should yield an object of the same shape as the $http service. This is an object that has a property named 'data'. The contact(s) returned from the service should be the value of that property.

Extra Credit

Add a new "column" to the contact. For example, a personal web page URL. All pages (except delete) should show the new column. If you added validation to create or update, then add validation for the new field.
